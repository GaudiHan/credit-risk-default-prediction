Key Findings
1
Debt burden is the strongest actionable flag. Applicants with a credit-to-income ratio above 3.5× are significantly more likely to default. This threshold is addressable at the point of application without model deployment.
2
External credit scores dominate prediction. EXT_SOURCE_2 and EXT_SOURCE_3 are the top two SHAP features. Applicants scoring below 0.3 on EXT_SOURCE_2 show a materially higher default rate than those above 0.6.
3
Thin-file applicants are a high-risk blind spot. Borrowers with no prior bureau history show an elevated default rate — yet are often approved on income alone. Alternative scoring signals are needed for this segment.
4
Employment tenure matters at the margin. Shorter employment duration (<1 year) correlates with higher default, particularly when combined with high annuity-to-income ratios.

Methodology
Data
307K applications + bureau history. Missing values imputed; class imbalance handled via scale_pos_weight.
Features
4 engineered features (debt burden, repayment affordability, employment stability, bureau exposure) added to 122 base features.
Model
XGBoost tuned via RandomizedSearchCV. Baseline: logistic regression. Evaluation: ROC-AUC on stratified holdout.
Explain
SHAP TreeExplainer used to surface per-feature impact on default probability — interpretable for non-technical stakeholders.
Recommendations
● Immediate
Introduce Credit-to-Income Hard Threshold
Flag applications with credit/income ratio above 3.5× for manual review. Implement as a pre-model rule to catch the highest-risk segment before scoring.
● Medium-Term
Deploy Model Score as Decision Supplement
Integrate XGBoost default probability as a second input alongside existing credit bureau checks. Target high-recall threshold to minimise missed defaults.
● Strategic
Build Alternative Scoring for Thin-File Applicants
Bureau-null applicants represent a growing segment. Explore telco, transactional, or employment verification data to reduce blind-spot defaults in this group.
